# OmniDev Kit

[English](README.md)

OmniDev Kit 是一个 AI 驱动的开发工作流工具包，将 AI 从「只会按指令敲代码的打字员」升级为**「懂成本控制、会做架构设计、能自己写测试、且永远不会忘事的高级研发工程师」**。

## 核心特色

### 1. 第一性原理 — 安全底线

两条根本性规则，统治所有 AI 行为，杜绝鲁莽执行：

- **需求判定与对齐（B.3）**：需求模糊、有歧义、关键信息缺失时，**禁止擅自解读、禁止自我发挥、禁止直接开始任务**。必须主动与用户确认核心诉求、最终目标、交付标准和本质问题后才可动手。
- **问题修复处理（B.4）**：接到 Bug 修复、漏洞整改、功能纠错类需求时，**禁止直接发补丁、禁止只给临时修复**。必须先输出完整解决方案（根因分析、影响范围、回归风险），多方案时择优并说明理由，待用户确认后才落地。

### 2. 国际化（i18n）

完整的中英文双语支持，运行时可切换：

- **`/od cfg -l zh`** / **`/od cfg -l en`** 切换语言。
- 阶段指令文件按 `phases/{locale}/` 和 `engine/{locale}/` 组织，**每次只加载一种语言**，不浪费 token。
- 所有用户可见输出（检查点、提示、报告、Q&A 菜单）自动适配当前语言。

### 3. 动态 Skill 组合（B.11）

OmniDev 不是孤立的单体工具，而是一个**编排器**，能动态发现和组合专业 Skill：

- **自动检测**用户输入中的排查/调试/修复意图关键词。
- **扫描本地 Skill**：项目级 + 用户级（`.cursor/skills/`、`.claude/skills/`、`.agents/skills/`）共 4 个目录。
- **分级匹配**：🎯 直接匹配（完整排查工作流）vs 🔧 辅助能力（日志查询、Pod 状态等）。
- **用户确认后才加载**（支持多选组合）。
- **无缝桥接**：外部 Skill 执行完毕后，可直接进入 OmniDev 开发流程修复问题。

### 4. 项目类型感知与自适应约束

- **历史项目**：AI 像「懂事的老员工」，100% 遵循现有规范，禁止强行引入 DDD/TDD。
- **全新项目**：全面启用现代软件工程规范 — 规范驱动开发、TDD/DDD、高覆盖率测试。
- `/od onboard` 扫描时自动识别 fullstack / frontend-only / backend-only / monorepo。

### 5. 智能自适应调度（T-Shirt Sizing）

- **S**：直接修复，跳过蓝图/计划。
- **M**：跳过蓝图 → 计划 → 开发 → 测试。
- **L/XL**：完整流程：蓝图 → 计划 → 开发 → 测试 → 部署。

### 6. 谋定而后动的工程纪律

- **强制脑暴**：禁止听到需求就写代码，必须先思考边界、异常和用户体验。
- **需求变更管理**（`/od ch`）：中途变更需求时自动输出影响评估，归档旧方案，生成新蓝图。
- **自动快照防错**：修改代码前强制 Git Commit 备份。

### 7. 跨会话记忆与状态持久化

- **双态存储**：`YAML Frontmatter + Markdown` — 机器精准读取，人类友好阅读。
- **记忆压缩**：状态文件超 200 行自动归档，防止幻觉和 token 浪费。
- **断点续传**：`/od resume` 从状态文件 + git 对比恢复上下文。

### 8. 懒加载上下文（B.7）

每个阶段通过 `context_requires` 声明所需文件，AI：
- 只加载声明的文件，跳过其他一切。
- 同会话内跨阶段缓存，不重复读取。
- 永远不预读下游阶段产物。
- Token 消耗与当前阶段实际需求成正比。

### 9. 交互模式与自动问答循环

- **交互模式**（默认开启）：所有决策点使用结构化选择 UI（AskQuestion），省去一轮对话。
- **自动问答循环**（默认开启）：每个 `/od` 命令执行完毕后，自动呈现上下文相关的下一步选项，而非静默停止。
- 通过 `/od cfg -i on|off` 一键开关。

### 10. DevSecOps 与韧性编码

Phase 3 强制执行安全与韧性编码：
- **安全设计**：IDOR/BOLA 防护、注入防护、SSRF/CSRF 防护、敏感数据脱敏。
- **标准级别**：结构化错误响应、超时控制、优雅降级、入口校验。
- **高级别**（用户指定）：熔断器、退避重试、舱壁隔离、优雅降级、限流。

### 11. 质量保证 — 测试（Phase 4）

- **依赖拓扑映射**：写测试前先梳理所有存储/三方/微服务依赖。
- **Mock 策略分级**：接口 Mock → 内存假实现 → 容器桩 → HTTP 桩 → MCP 驱动。
- **场景覆盖矩阵**：正常路径、校验、冲突、依赖故障、安全（IDOR/SQLi）、并发。
- **系统级韧性测试**：网络延迟、超时、高并发（P99 < 200ms）、内存压力。
- **覆盖率门槛**：>= 90% 语句/分支覆盖率。

### 12. 自我进化引擎

- **被动学习**：`/od` 会话中自动记录纠正、模式和错误修复。
- **智能提案**：信号积累到阈值时生成规则/技能改进提案。
- **用户掌控**：所有变更须通过 `/od ln` 明确批准。
- **安全护栏**：不能削弱核心规则，可通过 `/od ln --rb [N]` 回滚。

### 13. 企业汇报与运维

- **一键周报**（`/od rp`）：结合 git 历史 + 状态文件生成汇报级周报。
- **推送流程**（`/od ps`）：修改影响总结 → 暂存 → 生成提交信息 → 推送。
- **效能账单**：每次交付后 ROI 指标追加到 `metrics.json`。
- **手动更新**（`/od up`）：预览差异后确认才应用，绝不自动覆盖。

## 快速开始

**方式一：通过远程仓库地址安装（推荐）**

```
/od install https://github.com/zy-eagle/omnidev-kit.git
```

**方式二：从本地目录安装**

将 `INSTALL.md` 拖入 AI 助手对话框，说：「请帮我安装这个工具包」。

之后输入 `/od` 或直接提出需求即可开始。
